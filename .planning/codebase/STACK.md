## Build & Runtime Configuration

### Build System: Electron Vite

**Config File:** `electron.vite.config.ts`

Three separate build targets:

1. **Main Process** (`src/main/index.ts` → `dist/main/index.js`)
   - Electron main process (window management, IPC, backend)
   - Uses `externalizeDepsPlugin()` to exclude dependencies from bundle

2. **Preload Script** (`src/preload/index.ts` → `dist/preload/index.js`)
   - Context bridge for secure IPC between main and renderer
   - Exposes `CluiAPI` interface to renderer global scope
   - Uses `externalizeDepsPlugin()`

3. **Renderer** (`src/renderer/index.html` → `dist/renderer/`)
   - React application with Tailwind CSS
   - Plugins: React fast refresh (`@vitejs/plugin-react`), Tailwind CSS Vite plugin
   - Root HTML: `src/renderer/index.html`

### Scripts

```json
{
  "dev": "electron-vite dev",           // Local development
  "build": "electron-vite build",       // Production build
  "preview": "electron-vite preview",   // Preview production build
  "dist": "electron-vite build --mode production && electron-builder --mac --dir",
  "doctor": "bash scripts/doctor.sh",
  "postinstall": "electron-builder install-app-deps && bash scripts/patch-dev-icon.sh"
}
```

### Packaging Configuration

**electron-builder** (`package.json` "build" section):

- **Platform:** macOS only (currently)
- **Icon:** `resources/icon.icns`
- **Entitlements:** `resources/entitlements.mac.plist`
- **Microphone access:** Declared in `NSMicrophoneUsageDescription` (Whisper transcription)
- **Bundle:** Main + preload + renderer + resources copied to app

## Styling

### System: Tailwind CSS + PostCSS

**CSS Entry Point:** `src/renderer/index.css`

```css
@import "tailwindcss";
```

- Uses Tailwind v4 with `@tailwindcss/vite` plugin
- **PostCSS plugins:** Tailwind (via Vite plugin), autoprefixer
- **Global styles:** Base typography, scrollbar customization
- **Theme variables:** Runtime CSS custom properties (generated by `theme.ts`)

### Theme Architecture

**File:** `src/renderer/theme.ts`

- **Dual theme system:** Dark + light mode (runtime selection)
- **Color palette:** Derived from ChatCN oklch system
- **CSS custom properties:** `--clui-*` variables synced at runtime
- **Zustand store:** `useThemeStore()` manages theme state

**Key color tokens (dark mode sample):**
- Container: `#242422` (bg), `#3b3b36` (border)
- Accent: `#d97757` (orange)
- Text: `#ccc9c0` (primary), `#76766e` (tertiary)
- Status: running (`#d97757`), complete (`#7aac8c`), error (`#c47060`)

## Key Interfaces & Patterns

### IPC Bridge (Preload)

**File:** `src/preload/index.ts`

**CluiAPI interface exposes:**
- **Request-response:** `start()`, `createTab()`, `prompt()`, `cancel()`, `status()`, etc.
- **Session management:** `listSessions()`, `loadSession()`, `initSession()`
- **Media:** `attachFiles()`, `takeScreenshot()`, `pasteImage()`, `transcribeAudio()`
- **Window:** `resizeHeight()`, `setWindowWidth()`, `animateHeight()`, `hideWindow()`, `setIgnoreMouseEvents()`
- **Event listeners:** `onEvent()`, `onTabStatusChange()`, `onError()`, `onSkillStatus()`, `onWindowShown()`
- **Theme:** `getTheme()`, `onThemeChange()`
- **Plugins:** `fetchMarketplace()`, `installPlugin()`, `uninstallPlugin()`

All IPC channels defined in `src/shared/types.ts` enum `IPC`.

### Stream Event Types

**File:** `src/shared/types.ts`

- **InitEvent:** System initialization (session_id, tools, mcp_servers, model, etc.)
- **StreamEvent:** Normalized Claude Code streaming events
- **RateLimitEvent:** Rate limit responses
- **ResultEvent:** Task completion (success/error)
- **AssistantMessagePayload:** Message content with tool calls

### State Management

**sessionStore (Zustand):**
- Tabs: array of TabState (id, status, messages, attachments, workingDirectory)
- Active tab selection
- Static info: version, email, subscription, project path
- Model preference override
- Actions: addMessage, addAttachments, setTabStatus, etc.

**themeStore (Zustand):**
- Dark/light mode selection
- Expanded UI state
- System theme synchronization
